# ЗдравГарант — консольная медицинская система

Консольное приложение на Python, реализующее пользовательские сценарии
медицинской системы «ЗдравГарант» в точном соответствии с ранее разработанным
дизайном проекта.

## Связь с дизайном

Приложение воспроизводит:

- названия клиник, врачей и их специализации строго по дизайну;
- пользовательские сценарии: регистрация, вход, главная панель,
  запись на диагностику, история записей, настройки безопасности;
- возможность отменить любое действие на любом шаге.

## Архитектура

Проект построен на принципах ООП с разделением ответственности:

| Модуль | Класс / Содержимое | Роль |
|---|---|---|
| `person.py` | `Person` | Базовый класс (фамилия, имя, отчество, возраст) |
| `patient.py` | `Patient(Person)` | Пациент (пароль, email, список записей, авто-ID) |
| `doctor.py` | `Doctor(Person)` | Врач (специализация, клиника) |
| `clinic.py` | `Clinic` | Клиника (название, отделение) |
| `appointment.py` | `Appointment` | Запись на диагностику |
| `exceptions.py` | `CancelAction` | Исключение отмены действия |
| `validators.py` | `validate_password`, `validate_email` | Валидация пароля и email |
| `console_io.py` | `ConsoleIO` | Стратегия ввода/вывода |
| `storage.py` | `PickleStorage` | Стратегия хранения (pickle) |
| `medical_system.py` | `MedicalSystem` | Контейнер — бизнес-логика |
| `main.py` | — | Точка входа, текстовое меню |

### Ключевые решения

- **Наследование**: `Patient` и `Doctor` наследуют `Person`.
- **ФИО раздельно**: фамилия, имя и отчество хранятся в отдельных полях.
  Property `full_name` собирает их в строку без лишних пробелов.
- **Авто-генерация ID пациента**: формат PG-00001, PG-00002 и т.д.
  Пользователь не вводит ID — он назначается автоматически.
- **Email пациента**: обязательное уникальное поле. Используется как
  альтернативный логин при входе. `patient_id` остаётся внутренним
  идентификатором и ключом связи в `appointments`.
- **Вход по ID или email**: система принимает как `PG-00001`, так и email.
- **Валидация email**: базовая проверка (наличие @, домен с точкой, без пробелов).
  Реализована в `validators.py`.
- **Валидация пароля**: минимум 8 символов, строчная и заглавная латинские
  буквы, цифра, спецсимвол. Реализована в `validators.py`.
- **Подтверждение паролем**: изменение фамилии, имени, отчества, возраста
  и email в пользовательском режиме требует ввода текущего пароля.
- **Повтор пароля**: при несовпадении подтверждения пользователь возвращается
  на шаг ввода нового пароля (а не только подтверждения).
- **Стратегия ввода/вывода**: весь консольный ввод/вывод через `ConsoleIO`.
- **Административная панель**: стартовое меню включает просмотр, редактирование
  и удаление пациентов, а также сохранение/загрузку данных.
- **Стратегия хранения**: `PickleStorage` используется для сериализации данных
  в файл через административное меню.
- **Отмена действий**: ввод `cancel` / `q` / `quit` / `exit` на любом шаге
  прерывает текущую операцию через исключение `CancelAction`.
- **Меню через словарь функций**: обработка выбора без `if-elif` цепочек.

## Структура файлов

```
zdrav_garant/
├── person.py           — базовый класс Person
├── patient.py          — Patient(Person)
├── doctor.py           — Doctor(Person)
├── clinic.py           — Clinic
├── appointment.py      — Appointment
├── exceptions.py       — CancelAction
```

## Запуск

```bash
python main.py
```

Требуется Python 3.10+. Используется только стандартная библиотека.

## Пользовательские сценарии

### 1. Стартовое меню (административная панель)

```
--- Меню ---
  1. Регистрация пользователя
  2. Вход как пользователь
  3. Просмотр всех пользователей
  4. Редактирование пользователя
  5. Удаление пользователя
  6. Сохранение в файл
  7. Загрузка из файла
  8. Завершить программу
```

### 2. Панель пациента (после входа)

```
--- Меню ---
  1. Записаться на диагностику
  2. История записей
  3. Настройки безопасности
  4. Выйти из аккаунта
```

### 3. Подменю настроек безопасности

```
--- Меню ---
  1. Изменить фамилию
  2. Изменить имя
  3. Изменить отчество
  4. Изменить возраст
  5. Изменить email
  6. Изменить пароль
  7. Назад
```

### 4. Регистрация (авто-ID + email)

```
--- Регистрация пациента ---
  (для отмены введите cancel)
  Фамилия: Петров
  Имя: Иван
  Отчество (Enter — пропустить): Сергеевич
  Возраст: 30
  Email: petrov@mail.ru
  Введите пароль (мин. 8 символов, A-Z, a-z, 0-9, спецсимвол): MyPass1!x
  Повторите пароль: MyPass1!x
  [OK] Пациент 'Петров Иван Сергеевич' зарегистрирован.
  Ваш ID: PG-00001
  Ваш email: petrov@mail.ru
  Вход возможен по ID или по email.
```

### 5. Вход по ID

```
--- Вход в систему ---
  (для отмены введите cancel)
  Введите ID или email: PG-00001
  Введите пароль: MyPass1!x
  [OK] Добро пожаловать, Петров Иван Сергеевич!
```

### 6. Вход по email

```
--- Вход в систему ---
  (для отмены введите cancel)
  Введите ID или email: petrov@mail.ru
  Введите пароль: MyPass1!x
  [OK] Добро пожаловать, Петров Иван Сергеевич!
```

### 7. Просмотр всех пользователей

```
==================================================
  Все пациенты
==================================================
  [PG-00001] Петров Иван Сергеевич, возраст: 30, email: petrov@mail.ru
  [PG-00002] Кузнецова Мария Андреевна, возраст: 25, email: kuznetsova@yandex.ru
```

### 8. Редактирование пользователя (админ)

```
--- Редактирование пользователя (админ) ---
  (для отмены введите cancel)
  Введите ID пациента: PG-00001

--- Меню ---
  1. Изменить фамилию
  2. Изменить имя
  3. Изменить отчество
  4. Изменить возраст
  5. Изменить email
  6. Изменить пароль
  7. Назад

  Текущий email: petrov@mail.ru
  Новый email: petrov_new@mail.ru
  [OK] Email изменён на 'petrov_new@mail.ru'.
```

### 9. Изменение email пользователем

```
  Текущий email: petrov@mail.ru
  Введите новый email: petrov2@mail.ru
  Введите текущий пароль для подтверждения: MyPass1!x
  [OK] Email изменён на 'petrov2@mail.ru'.
```

### 10. Ошибка дублирования email

```
  Email: petrov@mail.ru
  [Ошибка] Этот email уже зарегистрирован. Введите другой.
  Email: ...
```

### 11. Удаление пользователя

```
--- Удаление пользователя ---
  (для отмены введите cancel)
  Введите ID пациента для удаления: PG-00002
  Будет удалён: [Пациент PG-00002] Кузнецова Мария Андреевна, возраст: 25, email: kuznetsova@yandex.ru
  Подтвердите удаление (да/нет): да
  [OK] Пациент Кузнецова Мария Андреевна (PG-00002) и все его записи удалены.
```

### 12. Сохранение в файл

При сохранении система запрашивает имя файла и расширение.
Оба поля необязательны — по умолчанию используется `zdrav_garant.pkl`.

```
--- Сохранение в файл ---
  (для отмены введите cancel)
  Имя файла (Enter — zdrav_garant):
  Расширение (Enter — pkl):
  [OK] Данные сохранены в 'zdrav_garant.pkl'.
```

Сохранение с пользовательским именем и расширением:

```
--- Сохранение в файл ---
  (для отмены введите cancel)
  Имя файла (Enter — zdrav_garant): backup
  Расширение (Enter — pkl): dat
  [OK] Данные сохранены в 'backup.dat'.
```

### 13. Загрузка из файла

При загрузке пользователь вводит полное имя файла целиком.
Ничего автоматически не достраивается.

```
--- Загрузка из файла ---
  (для отмены введите cancel)
  Введите полное имя файла: backup.dat
  [OK] Данные загружены из 'backup.dat'.
```

Если файл не найден:

```
--- Загрузка из файла ---
  (для отмены введите cancel)
  Введите полное имя файла: unknown.pkl
  [Ошибка] Файл 'unknown.pkl' не найден.
```

### 14. Отмена сохранения / загрузки

```
--- Сохранение в файл ---
  (для отмены введите cancel)
  Имя файла (Enter — zdrav_garant): cancel
  Сохранение отменено.
```

```
--- Загрузка из файла ---
  (для отмены введите cancel)
  Введите полное имя файла: cancel
  Загрузка отменена.
```

### 15. Отмена действия

```
--- Регистрация пациента ---
  (для отмены введите cancel)
  Фамилия: cancel
  Регистрация отменена.
```
