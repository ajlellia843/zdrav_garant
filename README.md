# ЗдравГарант — консольная медицинская система

Консольное приложение на Python, реализующее пользовательские сценарии
медицинской системы «ЗдравГарант» в точном соответствии с ранее разработанным
дизайном проекта.

## Связь с дизайном

Приложение воспроизводит:

- названия клиник, врачей и их специализации строго по дизайну;
- пользовательские сценарии: регистрация, вход, главная панель,
  запись на диагностику, история записей, настройки безопасности;
- возможность отменить любое действие на любом шаге.

## Архитектура

Проект построен на принципах ООП с разделением ответственности:

| Модуль | Класс / Содержимое | Роль |
|---|---|---|
| `person.py` | `Person` | Базовый класс (фамилия, имя, отчество, возраст) |
| `patient.py` | `Patient(Person)` | Пациент (пароль, список записей, авто-ID) |
| `doctor.py` | `Doctor(Person)` | Врач (специализация, клиника) |
| `clinic.py` | `Clinic` | Клиника (название, отделение) |
| `appointment.py` | `Appointment` | Запись на диагностику |
| `exceptions.py` | `CancelAction` | Исключение отмены действия |
| `validators.py` | `validate_password` | Валидация сложности пароля |
| `console_io.py` | `ConsoleIO` | Стратегия ввода/вывода |
| `storage.py` | `PickleStorage` | Стратегия хранения (pickle) |
| `medical_system.py` | `MedicalSystem` | Контейнер — бизнес-логика |
| `main.py` | — | Точка входа, текстовое меню |

### Ключевые решения

- **Наследование**: `Patient` и `Doctor` наследуют `Person`.
- **ФИО раздельно**: фамилия, имя и отчество хранятся в отдельных полях.
  Property `full_name` собирает их в строку без лишних пробелов.
- **Авто-генерация ID пациента**: формат PG-00001, PG-00002 и т.д.
  Пользователь не вводит ID — он назначается автоматически.
- **Валидация пароля**: минимум 8 символов, строчная и заглавная латинские
  буквы, цифра, спецсимвол. Реализована в `validators.py`.
- **Подтверждение паролем**: изменение фамилии, имени, отчества и возраста
  требует ввода текущего пароля.
- **Повтор пароля**: при несовпадении подтверждения пользователь возвращается
  на шаг ввода нового пароля (а не только подтверждения).
- **Стратегия ввода/вывода**: весь консольный ввод/вывод через `ConsoleIO`.
- **Административная панель**: стартовое меню включает просмотр, редактирование
  и удаление пациентов, а также сохранение/загрузку данных.
- **Стратегия хранения**: `PickleStorage` используется для сериализации данных
  в файл через административное меню.
- **Отмена действий**: ввод `cancel` / `q` / `quit` / `exit` на любом шаге
  прерывает текущую операцию через исключение `CancelAction`.
- **Меню через словарь функций**: обработка выбора без `if-elif` цепочек.

├── appointment.py      — Appointment

## Запуск

```bash
python main.py
```

Требуется Python 3.10+. Используется только стандартная библиотека.

## Пользовательские сценарии

### 1. Стартовое меню (административная панель)

```
--- Меню ---
  1. Регистрация пользователя
  2. Вход как пользователь
  3. Просмотр всех пользователей
  4. Редактирование пользователя
  5. Удаление пользователя
  6. Сохранение в файл
  7. Загрузка из файла
  8. Завершить программу
```

### 2. Панель пациента (после входа)

```
--- Меню ---
  1. Записаться на диагностику
  2. История записей
  3. Настройки безопасности
  4. Выйти из аккаунта
```

### 3. Подменю настроек безопасности

```
--- Меню ---
  1. Изменить фамилию
  2. Изменить имя
  3. Изменить отчество
  4. Изменить возраст
  5. Изменить пароль
  6. Назад
```

### 4. Регистрация (авто-ID)

```
--- Регистрация пациента ---
  (для отмены введите cancel)
  Фамилия: Петров
  Имя: Иван
  Отчество (Enter — пропустить): Сергеевич
  Возраст: 30
  Введите пароль (мин. 8 символов, A-Z, a-z, 0-9, спецсимвол): MyPass1!x
  Повторите пароль: MyPass1!x
  [OK] Пациент 'Петров Иван Сергеевич' зарегистрирован.
  Ваш ID: PG-00001
  Запомните его для входа в систему.
```

### 5. Просмотр всех пользователей

```
=== Все пациенты ===
  1. [PG-00001] Петров Иван Сергеевич, возраст: 30
  2. [PG-00002] Кузнецова Мария Андреевна, возраст: 25
```

### 6. Редактирование пользователя (админ)

```
--- Редактирование пользователя (админ) ---
  (для отмены введите cancel)
  Введите ID пациента: PG-00001
  Выбран: [PG-00001] Петров Иван Сергеевич, 30

--- Меню ---
  1. Изменить фамилию
  2. Изменить имя
  3. Изменить отчество
  4. Изменить возраст
  5. Изменить пароль
  6. Назад

  Текущая фамилия: Петров
  Новая фамилия: Сидоров
  [OK] Фамилия изменена на 'Сидоров'.
```

### 7. Удаление пользователя

```
--- Удаление пользователя ---
  (для отмены введите cancel)
  Введите ID пациента для удаления: PG-00002
  Будет удалён: [PG-00002] Кузнецова Мария Андреевна, 25
  Подтвердите удаление (да/нет): да
  [OK] Пациент Кузнецова Мария Андреевна (PG-00002) и все его записи удалены.
```

### 8. Сохранение и загрузка

```
  [OK] Данные сохранены в zdrav_garant.pkl
  [OK] Данные загружены из zdrav_garant.pkl
```

### 9. Ошибка подтверждения пароля

```
  Введите пароль (мин. 8 символов, A-Z, a-z, 0-9, спецсимвол): MyPass1!x
  Повторите пароль: другой
  [Ошибка] Пароли не совпадают. Введите пароль заново.
  Введите пароль (мин. 8 символов, A-Z, a-z, 0-9, спецсимвол): ...
```

### 10. Отмена действия

```
--- Регистрация пациента ---
  (для отмены введите cancel)
  Фамилия: cancel
  Регистрация отменена.
```
