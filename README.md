# ЗдравГарант — консольная медицинская система

Консольное приложение на Python, реализующее пользовательские сценарии
медицинской системы «ЗдравГарант» в точном соответствии с ранее разработанным
дизайном проекта.

## Связь с дизайном

Приложение воспроизводит:

- названия клиник, врачей и их специализации строго по дизайну;
- пользовательские сценарии: регистрация, вход, главная панель,
  запись на диагностику, история записей, настройки безопасности;
- возможность отменить любое действие на любом шаге.

## Архитектура

Проект построен на принципах ООП с разделением ответственности:

| Модуль | Класс | Роль |
|---|---|---|
| `person.py` | `Person` | Базовый класс (id, ФИО, возраст) |
| `patient.py` | `Patient(Person)` | Пациент (пароль, список записей) |
| `doctor.py` | `Doctor(Person)` | Врач (специализация, клиника) |
| `clinic.py` | `Clinic` | Клиника (название, отделение) |
| `appointment.py` | `Appointment` | Запись на диагностику |
| `exceptions.py` | `CancelAction` | Исключение отмены действия |
| `console_io.py` | `ConsoleIO` | Стратегия ввода/вывода |
| `storage.py` | `PickleStorage` | Стратегия хранения (pickle) |
| `medical_system.py` | `MedicalSystem` | Контейнер — бизнес-логика |
| `main.py` | — | Точка входа, текстовое меню |

### Ключевые решения

- **Наследование**: `Patient` и `Doctor` наследуют `Person`.
- **Стратегия ввода/вывода**: весь консольный ввод/вывод проходит через `ConsoleIO`.
- **Стратегия хранения**: сериализация через `PickleStorage` (модуль `pickle`).
- **Контейнерный класс**: `MedicalSystem` агрегирует все сущности и логику.
- **Меню через словарь функций**: обработка выбора без `if-elif` цепочек.
- **Отмена действий**: ввод `cancel` / `q` / `quit` / `exit` на любом шаге
  прерывает текущую операцию через исключение `CancelAction`.
- **Валидация даты**: формат ДД.ММ.ГГГГ, реальная календарная дата,
  не раньше сегодняшнего дня.
- **Безопасность паролей**: минимум 8 символов, подтверждение при регистрации
  и смене пароля, проверка текущего пароля при смене.

## Запуск

```bash
python main.py
```

Требуется Python 3.10+. Используется только стандартная библиотека.

## Пользовательские сценарии

### 1. Регистрация

```
--- Меню ---
  1. Регистрация
  2. Вход
  3. Выход
Выберите пункт: 1

--- Регистрация пациента ---
  (для отмены введите cancel)
  Введите ID: 1
  Введите ФИО: Петров Иван Сергеевич
  Введите возраст: 30
  Введите пароль (мин. 8 символов): mypass123
  Повторите пароль: mypass123
  [OK] Пациент 'Петров Иван Сергеевич' зарегистрирован.
```

### 2. Запись на диагностику

```
--- Запись на диагностику ---
  (для отмены введите cancel)

==================================================
  Список клиник
==================================================
  [Клиника ID: 1] Городская клиническая больница №1, отделение: Первичное отделение
  [Клиника ID: 2] Центр здоровья «Пульс», отделение: Первичное отделение
  [Клиника ID: 3] Клиника современной медицины, отделение: Первичное отделение
  Введите ID клиники: 2

==================================================
  Врачи выбранной клиники
==================================================
  [Врач ID: 3] Сидорова Анна Сергеевна, специализация: Невролог
  [Врач ID: 4] Кузнецов Олег Викторович, специализация: Хирург
  Введите ID врача: 3
  Введите дату приёма (ДД.ММ.ГГГГ): 15.04.2026
  [OK] Запись #1 создана: Сидорова Анна Сергеевна, Центр здоровья «Пульс», 15.04.2026
```

### 3. Отмена действия

```
--- Регистрация пациента ---
  (для отмены введите cancel)
  Введите ID: cancel
  Регистрация отменена.
```

### 4. Смена пароля

```
--- Настройки безопасности (Петров Иван Сергеевич) ---

--- Меню ---
  1. Изменить ФИО
  2. Изменить возраст
  3. Изменить пароль
  4. Назад
Выберите пункт: 3

--- Смена пароля ---
  (для отмены введите cancel)
  Введите текущий пароль: mypass123
  Введите новый пароль (мин. 8 символов): newpass456
  Повторите новый пароль: newpass456
  [OK] Пароль успешно изменён.
```
